## MODULE 1: PROPOSITIONAL LOGIC OPERATORS

### Key Concepts
* **CONJUNCTION (AND / ∧):** True only if both statements are True.
* **DISJUNCTION (OR / ∨):** True if at least one statement is True.
* **BICONDITIONAL (IF AND ONLY IF / ↔):** True only if both statements have the exact same truth value (T ↔ T or F ↔ F).
* **IMPLICATION (IF-THEN / →):** This evaluates to False in **ONLY ONE** scenario: when a True premise leads to a False conclusion (T → F = F). In all other cases, it is mathematically True.
  * *Equivalence Rule:* P → Q is logically equivalent to ¬P ∨ Q.

### The Implication Trap (P → Q)
Think of P → Q as a strict database business rule: *"If a user is an Admin (P), then they must have Write Access (Q)."*
* **Admin WITH Write Access (T → T):** Valid state. (True)
* **Admin WITHOUT Write Access (T → F):** **SYSTEM ERROR.** The condition is met, but the requirement is missing. This is the ONLY time the rule evaluates to False.
* **Standard User WITH Write Access (F → T):** Valid state. The rule only explicitly restricts Admins; it doesn't forbid standard users from having write access. (True)
* **Standard User WITHOUT Write Access (F → F):** Valid state. (True)

## MODULE 2: LOGICAL EQUIVALENCES & TRUTH TABLES

### Key Concepts
* **DE MORGAN'S LAWS:** Distributing a negation (¬) flips the internal operators.
  * ¬(P ∨ Q) ≡ ¬P ∧ ¬Q
  * ¬(P ∧ Q) ≡ ¬P ∨ ¬Q
* **DISTRIBUTIVE LAWS:** Similar to algebraic multiplication.
  * P ∧ (Q ∨ R) ≡ (P ∧ Q) ∨ (P ∧ R)
  * P ∨ (Q ∧ R) ≡ (P ∨ Q) ∧ (P ∨ R)

## MODULE 3: TAUTOLOGY, CONTRADICTION, AND CONTINGENCY

### Key Concepts
* **TAUTOLOGY:** A compound proposition that is always True, regardless of the individual truth values of its variables. (e.g., P ∨ ¬P).
* **CONTRADICTION:** A compound proposition that is always False. (e.g., P ∧ ¬P).
* **CONTINGENCY:** A proposition that is neither a tautology nor a contradiction; its truth value depends on the inputs.

## MODULE 4: FIRST-ORDER LOGIC (QUANTIFIERS)

### Key Concepts
* **UNIVERSAL QUANTIFIER (∀x):** "For all x". The rule applies to every single element in the domain.
* **EXISTENTIAL QUANTIFIER (∃x):** "There exists an x". The rule applies to at least one element in the domain.
* **NEGATING QUANTIFIERS:** * Negating a "For All" creates an "Exists": ¬(∀x, P(x)) ≡ ∃x, ¬P(x)
  * Negating an "Exists" creates a "For All": ¬(∃x, P(x)) ≡ ∀x, ¬P(x)
* **THE IMPLICATION NEGATION TRAP:** * How to negate ∀x (P(x) → Q(x))
  * Result: **∃x (P(x) ∧ ¬Q(x))**
  * *Logic:* To disprove "All P's are Q's", you do not say "No P's are Q's". You just need to find "At least one P that is NOT Q".

## DAILY VERIFICATION Q&A (COMBINED QUESTION BANK)

**Q1. Which of the following is the truth value of (P → Q) when P is False and Q is True?**
* **Answer:** True 
* **Logic:** An implication is only false if True leads to False. False leading to True does not violate the implication rule.

**Q2. According to De Morgan's Law, the expression ¬(P ∨ Q) is logically equivalent to:**
* **Answer:** ¬P ∧ ¬Q 
* **Logic:** The negation distributes to both variables, and the OR (∨) flips to an AND (∧).

**Q3. The compound proposition P ∨ ¬(P ∧ Q) is a:**
* **Answer:** Tautology 
* **Logic:** By De Morgan's, this becomes P ∨ (¬P ∨ ¬Q). Grouping the P's gives (P ∨ ¬P) ∨ ¬Q. Since (P ∨ ¬P) is always True, the entire statement evaluates to True regardless of Q.

**Q4. What is the negation of the statement: "All data engineers use SQL" (∀x, P(x))?**
* **Answer:** There exists at least one data engineer who does not use SQL. 
* **Logic:** The negation of a Universal ("All") is an Existential ("At least one"), combined with the negation of the action.

**Q5. If P is True and Q is False, what is the truth value of (¬P ∨ Q)?**
* **Answer:** False 
* **Logic:** ¬(True) ∨ False becomes False ∨ False, which evaluates to False.

**Q6. Is ¬(P ∧ Q) equivalent to (¬P ∨ ¬Q)?**
* **Answer:** Yes 
* **Logic:** This is the exact definition of De Morgan's Law.

**Q7. Is (P ∧ ¬P) a Tautology or a Contradiction?**
* **Answer:** Contradiction 
* **Logic:** A statement cannot be simultaneously True AND False. The result is always False.

**Q8. Negate the statement: ∀x (P(x) → Q(x)).**
* **Answer:** ∃x (P(x) ∧ ¬Q(x))
* **Logic:** Flip the ∀ to an ∃. Then convert P → Q to ¬P ∨ Q. Negating that using De Morgan's Law results in P ∧ ¬Q.
